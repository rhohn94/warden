# Feature manifest

This file is the source of truth for the **sync adoption phase** (F2). It
catalogs every Grimoire framework feature that has an idempotent adoption step —
features an old project might not have enabled yet and that `sync-from-upstream`
should offer (or auto-run under Noir) when the project syncs to a newer release.

Each entry records how to detect whether the feature is already adopted
(`detect`), how to enable it idempotently (`adopt`), and — when user data must
move — an optional explicitly-confirmed migration step (`migrate`). The
`introduced-in` version drives delta computation: features older than the
project's `framework-version` marker are skipped without running `detect`.

**Format:** versioned Markdown table, same pattern as `baseline-requirements.md`.
Bump `manifest-version` whenever a row is **added** or a row's schema changes,
so tooling can detect and merge new entries idempotently.
