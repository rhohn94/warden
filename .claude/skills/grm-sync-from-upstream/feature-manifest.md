# Feature manifest

This file is the source of truth for the **sync adoption phase** (F2). It
catalogs every Grimoire framework feature that has an idempotent adoption step â€”
features an old project might not have enabled yet and that `grm-sync-from-upstream`
should offer (or auto-run under Noir) when the project syncs to a newer release.

Each entry records how to detect whether the feature is already adopted
(`detect`), how to enable it idempotently (`adopt`), and â€” when user data must
move â€” an optional explicitly-confirmed migration step (`migrate`). The
`introduced-in` version drives delta computation: features older than the
project's `framework-version` marker are skipped without running `detect`.

**Format:** versioned Markdown table, same pattern as `baseline-requirements.md`.
Bump `manifest-version` whenever a row is **added** or a row's schema changes,
so tooling can detect and merge new entries idempotently.

**Detect-predicate rule (enforced).** A `detect` predicate must reference only
artifacts that are actually **distributed** to a consumer â€” skills, scripts,
config, gate text. It must **never** depend on a path the v3.39 Bulkhead excludes
from the sync/build (anything under `docs/grimoire/design/` and the other
`docs/grimoire/` framework-internal docs): a consumer never receives those, so the
detect can never pass there and the adoption is silently skipped every sync. The
`grm-doc-assurance manifest-detect-hygiene` check enforces this at release closeout.
Cite an internal design doc in `summary`/`adopt` prose if useful â€” just not in
`detect`.
